# py-streamlit

전문적인 Streamlit 데이터 앱 개발을 위한 스킬입니다. 사용자가 Streamlit 앱 개발, 데이터 대시보드 구축, 시각화 도구 생성을 요청할 때 이 스킬을 사용하세요. 시각화에는 반드시 **Plotly**만을 사용하며, 데이터 분석가가 실무에서 즉시 사용할 수 있는 수준의 UX/UI, 분석 정밀도, 그리고 기술적 최적화를 보장합니다.

## 트리거 조건
- Streamlit 대시보드나 앱을 만들어달라고 할 때
- 기존 Streamlit 코드를 개선하거나 최적화해달라고 할 때
- 파이썬 데이터를 웹 대시보드로 시각화하고 싶어할 때
- 데이터 분석 분석 결과를 인터랙티브하게 공유하고 싶어할 때

## 핵심 가이드라인

### 1. 사용자 경험 및 인터페이스 (UX/UI)
- **반응형 레이아웃:** `st.columns`를 활용하여 브라우저 크기에 따라 위젯과 차트가 유연하게 재배치되도록 구성합니다.
- **사이드바 활용:** 필터와 설정값들은 `st.sidebar`에 정리하여 메인 콘텐츠 영역을 확보합니다.
- **직관적인 위젯:** 범주형 데이터에는 `st.selectbox`, 다중 선택에는 `st.multiselect`, 범위 선택에는 `st.slider` 등 용도에 맞는 위젯을 사용합니다.
- **로딩 상태 표시:** 데이터 로딩이나 복잡한 계산 시 `st.spinner` 또는 `st.progress`를 사용하여 사용자에게 진행 상황을 알립니다.
- **Plotly 상호작용:** 모든 차트는 Plotly를 사용하며, 줌, 툴팁, 데이터 추출 기능이 활성화된 인터랙티브한 차트를 제공합니다.
- **색상 및 폰트:** 데이터의 의미(예: 긍정-녹색, 부정-적색)에 맞는 컬러 팔레트를 유지하고, 가독성 높은 폰트 크기와 굵기를 적용합니다.
- **빈 화면 처리:** 필터 결과가 없을 경우 `st.info("데이터가 없습니다")`와 같은 안내 문구를 표시합니다.
- **도움말 제공:** `help` 파라미터를 통해 KPI 지표나 필터의 정의를 상세히 설명합니다.
- **페이지 내비게이션:** 분석 목적에 따라 `st.navigation` 기능을 적절히 활용하여 멀티 페이지 구조를 구현합니다.

### 2. 데이터 분석 및 시각화 정밀도
- **KPI 요약:** 대시보드 상단에 가장 중요한 지표들을 `st.metric`으로 요약하여 배치합니다.
- **필터 동기화:** 사이드바 필터 변경이 모든 차트와 테이블에 즉각 반영되도록 로직을 구성합니다.
- **데이터 드릴다운:** `st.tabs`를 활용하여 요약 정보와 상세 데이터를 구분해 탐색할 수 있게 합니다.
- **축의 적절성:** 차트의 Y축이 0부터 시작하는지 확인하여 데이터 왜곡을 방지합니다.
- **이상치 처리:** 데이터 내 극단값이 시각화를 방해할 때 이를 제어할 수 있는 옵션(예: 슬라이더로 범위 제한)을 제공합니다.
- **단위 명시:** 숫자 데이터에 단위(%, 원, $, 건)를 누락 없이 표기합니다.
- **비교 기준 존재:** 현재 수치가 전일/전월 대비 높은지 낮은지 비교 지표(delta)를 포함합니다.
- **데이터 다운로드:** 분석된 결과(CSV/Excel)를 사용자가 직접 내려받을 수 있는 `st.download_button`을 제공합니다.
- **정렬 기능:** `st.dataframe`을 사용하여 컬럼별 정렬 및 검색이 원활하도록 합니다.

### 3. 기술적 최적화 및 성능 (Engineering)
- **캐싱 전략:** `@st.cache_data`나 `@st.cache_resource`를 사용하여 데이터 로딩 및 무거운 연산 속도를 최적화합니다.
- **쿼리 최적화:** DB 연결 시 필요한 데이터만 가져오도록 최적화된 쿼리를 작성합니다.
- **에러 핸들링:** 데이터 부재나 서버 오류 시 `st.error`나 `st.exception`을 통해 친절하게 안내합니다.
- **세션 상태 관리:** `st.session_state`를 활용해 페이지 전환 시에도 사용자 설정값이 유지되도록 합니다.
- **종속성 관리:** `requirements.txt`에 라이브러리 버전과 함께 필요한 패키지를 명확히 기재합니다.
- **코드 모듈화:** 데이터 전처리, 시각화 로직을 별도 함수나 파일로 분리하여 유지보수가 용이하게 합니다.
- **메모리 효율성:** 대용량 데이터프레임을 다룰 때 불필요한 복사본 생성을 지양합니다.
- **보안:** API 키나 DB 접속 정보가 `.streamlit/secrets.toml`을 통해 안전하게 관리되도록 안내합니다.

## 권장 프로젝트 구조
```text
project_root/
├── app.py                # 메인 실행 파일
├── pages/                # 추가 페이지 파일들
├── src/
│   ├── data_loader.py    # 데이터 로딩 및 캐싱 로직
│   ├── processor.py      # 데이터 처리 및 전처리 함수
│   └── charts.py         # Plotly 시각화 생성 함수
├── .streamlit/
│   └── secrets.toml      # 보안 설정 및 시크릿
└── requirements.txt      # 패키지 의존성 파일
```
